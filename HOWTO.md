## 1. Architecture overview

```
GeneratePress (parent)          — unmodified shell, never touch
  └─ generatepress-child/       — brand chrome only (fonts, GP resets, header, CSS tokens)
       style.css                — all custom CSS; no Customizer CSS, no <style> tags
       functions.php            — enqueues style.css only
       page.php                 — calls mlk_render_sections() for every page

wp-content/plugins/
  mudlark-acf/                  — the build plugin; owns ALL layout logic
    mudlark-acf.php             — bootstrapper; loads fields + render, wires hooks
    fields.php                  — registers ACF flexible-content field group (PHP, no JSON)
    render.php                  — one function per section layout; outputs HTML
    data-populate.php           — one-shot seeder; run via ?mlk_populate=1 in WP admin
  mudlark-cpt/                  — product CPT, artist taxonomy, PayPal meta (separate concern)
```

**Rules that must never be broken:**
- No `<style>` tags in PHP. No Customizer Additional CSS. No inline `style=""` on layout elements. All CSS lives in `style.css`.
- GP is untouched — no GP Customizer changes that persist in the DB. Everything is done in the child theme.
- No Gutenberg / GenerateBlocks block HTML generated by hand. This site uses ACF flexible content, not blocks.

## 4. How sections work

Every page uses `page.php` as its template:

```php
mlk_render_sections( get_the_ID() );
```

This reads the `page_sections` ACF flexible content field for that page and calls the matching renderer for each row.

### Section layout names → renderer functions

| ACF layout name | Renderer | Description |
|---|---|---|
| `hero` | `mlk_s_hero()` | Full-bleed photo or solid-colour hero with title overlay |
| `page_banner` | `mlk_s_banner()` | Compact coloured banner for inner pages |
| `intro_text` | `mlk_s_intro()` | Centred heading + body + optional button |
| `tinted_band` | `mlk_s_band()` | Coloured band with heading, body, button |
| `two_column` | `mlk_s_twocol()` | Photo left/right + text block; configurable side + tint |
| `artist_grid` | `mlk_s_artists()` | 3-col card grid with photos, names, bios |
| `workshop_list` | `mlk_s_workshops()` | List of workshop items with dates + book buttons |
| `ticket_tailor` | `mlk_s_tickets()` | Ticket Tailor inline widget embed |
| `info_grid` | `mlk_s_info_grid()` | 2-col icon/heading/body grid (plan-your-visit style) |
| `contact_cards` | `mlk_s_contact_cards()` | 3-col icon cards (contact page) |
| `contact_form` | `mlk_s_contact_form()` | WPForms embed by form ID |
| `mailing_signup` | `mlk_s_mailing()` | Email signup band |

### Section HTML pattern

Every section follows the same structure:

```html
<section class="mlk-[name] mlk-section [modifiers]">
    <div class="mlk-inner">
        <!-- content here -->
    </div>
</section>
```

- **`mlk-section`** — base class (no styles, used for targeting)
- **`.mlk-inner`** — `max-width: 1100px; margin: 0 auto; padding: 0 2.5rem` — constrains content width
- The **`<section>`** itself is always full-bleed (edge to edge). Background colour lives on the section. Padding is vertical only (`padding: Nrem 0`).

**Exceptions to the inner pattern:**
- `mlk-hero` and `mlk-page-banner` are intentionally full-bleed with no `.mlk-inner` (the hero is a positioned overlay layout, not a content flow).
- `mlk-twocol .mlk-inner` uses `display: grid` rather than being a plain content wrapper.

## 6. Brand tokens & CSS architecture

All tokens are CSS custom properties in `:root` in `style.css`:

```css
--ink:       #2c2318  /* near-black — headings, logo */
--chalk:     #f5f0e8  /* warm white — header bg, text on dark */
--bg:        #e8e2d8  /* page background (warm greige) */
--mud:       #335a70  /* teal/steel — PRIMARY brand colour */
--reed:      #6d8b85  /* muted sage */
--rust:      #bc9c22  /* gold — CTA accent */
--pink:      #b07070  /* dusty rose */
--neon:      #e8ff00  /* neon yellow — squiggle only */
```

**Band tints** (for `bg_tint` choices):
```css
--reed-band: #e9eeed  /* light sage */
--blue-band: #e0e6ea  /* light teal */
--pink-band: #f3eaea  /* light rose */
--chalk-band:#f5f0e8  /* warm white */
--ink-band:  #2c2318  /* near-black */
```

### Adding a new tint option

1. Add the CSS variable to `:root` if needed
2. Add a CSS rule: `.mlk-[section]--[tint] { background: var(--[tint]-band); }`
3. Add the choice to the ACF field in `fields.php`

### The squiggle

Every section heading uses `mlk_squiggle()` immediately after the `<h2>`. It outputs a fixed SVG brushstroke in `--neon`. Don't replace it with a CSS border — it's intentionally handmade-looking.

## 7. Adding a new section type

**Step 1 — `fields.php`:** Add a new layout array inside the `layouts` array of the flexible content field. Follow the existing pattern exactly. Use a unique `field_mlk_` prefixed key for every field.

```php
[
    'key'    => 'layout_mlk_myblock',
    'name'   => 'my_block',
    'label'  => 'My Block',
    'sub_fields' => [
        [ 'key' => 'field_mlk_mb_heading', 'name' => 'heading', 'label' => 'Heading', 'type' => 'text' ],
        // ...
    ],
],
```

**Step 2 — `render.php`:** Add a case in the `switch` statement in `mlk_render_sections()`:

```php
case 'my_block': mlk_s_myblock( $row ); break;
```

Then write the renderer:

```php
function mlk_s_myblock( array $r ): void {
    ?>
    <section class="mlk-myblock mlk-section">
        <div class="mlk-inner">
            <h2><?php echo mlk_t( $r['heading'] ?? '' ); ?></h2>
        </div>
    </section>
    <?php
}
```

**Step 3 — `style.css`:** Add section styles. Always: vertical padding on the section, no horizontal padding (`.mlk-inner` handles that):

```css
.mlk-myblock { padding: 6rem 0; background: var(--bg); }
```

**Step 4 — `data-populate.php`** (optional): If you want this section to appear on a page when seeding, add it to the relevant page's `$sections` array.

## 8. Artist pages

### Page structure (each artist page)

| Section index | Layout | Purpose |
|---|---|---|
| 0 | `hero` | Name + kicker + subtitle overlay; leave image empty for solid colour, add art montage when ready |
| 1 | `two_column` | Artist portrait photo (right or left) + bio text |
| 2+ | Additional sections as needed |

### Moving / setting artist photos via DB

ACF stores images as attachment IDs. The meta key pattern is:

```
page_sections_{index}_{field_name}         → attachment ID (integer as string)
_page_sections_{index}_{field_name}        → ACF field key (e.g. field_mlk_tc_image)
```

To set the hero background image on an artist page:
```sql
UPDATE wp_postmeta SET meta_value = '{attachment_id}' WHERE post_id = {page_id} AND meta_key = 'page_sections_0_image';
UPDATE wp_postmeta SET meta_value = 'field_mlk_hero_image' WHERE post_id = {page_id} AND meta_key = '_page_sections_0_image';
```

To set the two-col image (section index 1):
```sql
UPDATE wp_postmeta SET meta_value = '{attachment_id}' WHERE post_id = {page_id} AND meta_key = 'page_sections_1_image';
UPDATE wp_postmeta SET meta_value = 'field_mlk_tc_image' WHERE post_id = {page_id} AND meta_key = '_page_sections_1_image';
```

## 9. Data-populate

`data-populate.php` seeds all page content from scratch. Trigger it by visiting:

```
http://mudlark-acf.local/reindeer-stable/?mlk_populate=1
```

(Requires `manage_options` capability — you must be logged in as an admin.)

**What it does:**
- Creates/updates pages (home, artists, what's on, special events, plan your visit, contact, katie, caroline, hanna)
- Sets the `page_sections` ACF flexible content for each page with realistic placeholder content
- Sets page parents, slugs, and menu order
- Sets the static front page

**When to re-run:** Only if you've wiped the DB or are setting up a fresh Local site. Running it again on an existing site will overwrite ACF content on those pages.

**Adding a page to the seeder:**

In `mlk_populate_all()`, follow the existing `mlk_upsert_page()` + `mlk_set_sections()` pattern. Each section is an array matching the ACF flexible content structure.

## 10. ACF meta key conventions

ACF stores flexible content in `wp_postmeta` with a predictable naming pattern:

```
page_sections                          → serialised layout-name array (e.g. ["hero","two_column"])
page_sections_0_acf_fc_layout         → layout name of section 0
page_sections_0_{field_name}          → value for that field in section 0
_page_sections_0_{field_name}         → ACF field key reference (e.g. field_mlk_hero_image)

page_sections_1_acf_fc_layout         → layout name of section 1
page_sections_1_{field_name}          → value for section 1
...

page_sections_1_artists               → repeater count (integer)
page_sections_1_artists_0_{field}     → first repeater row, field value
_page_sections_1_artists_0_{field}    → first repeater row, field key reference
```

**Field key reference** (the underscore-prefixed meta) must match the key registered in `fields.php`. If it's wrong, ACF will show the value as empty in the admin even though the raw value exists in the DB.

**Image fields** store the attachment ID as a plain integer string. The renderer calls `get_field()` which returns a full image array (url, alt, width, height etc.) because `return_format => 'array'` is set in `fields.php`.

## 11. Header & navigation

### Layout

The sticky header is `64px` tall on a `--chalk` (#f5f0e8) background. Inside it:

```
.site-header
  .inside-header   [display:flex; justify-content:space-between; padding: 0 2.5rem]
    .site-branding  [logo — left]
    #site-navigation [nav — pushed right by GP's margin-left:auto]
```

GP's `nav-float-right` body class normally adds `margin-left: auto` to `#site-navigation`, which we allow (this is what pushes the nav to the right).

### Logo

The logo is the site title rendered as text in `Amatic SC` font. It's not an image file. Styled in `style.css`:

```css
.site-header .site-branding .site-title a {
  font-family: 'Amatic SC', cursive;
  font-weight: 700;
  font-size: 1.75rem;
  color: var(--ink);
}
```

The neon squiggle underline under the logo is a CSS `::after` pseudo-element — check `style.css` for the current implementation.

### Navigation items

Styled via GP's nav classes. The "Book a Workshop" item gets a gold pill treatment via the `menu-item-cta` class on that menu item in WP Admin → Appearance → Menus.

### Mobile hamburger — GP-first pattern

**Never fight GP with CSS `display` overrides.** The correct approach:

**1. Set the breakpoint via PHP filter in `functions.php`:**
```php
add_filter( 'generate_mobile_menu_media_query', function () {
    return '(max-width: 1100px)';
} );
```
This is the official GP hook. It replaces GP's default `768px` hamburger breakpoint without GP Premium. The CSS `@media (max-width: 1100px)` in `style.css` must match this value.

**2. GP Customizer → Layout → Navigation → Drop Point: set to 0 (disabled).**
The Drop Point with Float Right creates an *intermediate stacked layout* (not a hamburger) between 769px and the drop point. With the filter doing the work, the drop point must be cleared.

**3. Style the toggle button — sizing only, no `display` overrides:**
```css
.menu-toggle {
  border: 1px solid var(--ink);
  border-radius: 3px;
  padding: 6px 10px;
  background: transparent;
  line-height: 1;
  color: var(--ink);
}
/* Size the SVG icons only — never set display:block here.
   GP renders TWO SVGs inside .gp-icon (≡ open + × close) and toggles
   them via its own CSS. Setting display:block shows both simultaneously. */
.menu-toggle .gp-icon svg {
  width: 20px;
  height: 20px;
  fill: var(--ink);
}
```

**4. Drawer styles — scoped to `.toggled` only:**
```css
@media (max-width: 1100px) {
  #site-navigation.toggled {
    background: #fafaf8 !important;   /* off-white — clearly floats over content */
    border-top: 1px solid var(--border);
    border-bottom: 2px solid var(--border);
    box-shadow: 0 6px 18px rgba(0,0,0,0.12);
    position: absolute;
    top: 100%;
    left: 0; right: 0;
    z-index: 200;
  }
  /* Hide GP's internal duplicate toggle */
  #site-navigation .menu-toggle { display: none !important; }
  /* Nav list — target ul.sf-menu via div.main-nav, NOT ul.main-nav (wrong selector) */
  #site-navigation.toggled .main-nav ul { ... }
}
```

**Key DOM facts:**
- GP renders a *duplicate* `.menu-toggle` inside `#site-navigation` — always `display: none !important` it.
- The external toggle is in `.mobile-menu-control-wrapper` (shown/hidden by GP at the breakpoint).
- GP's nav UL class is `sf-menu`. The wrapper div is `main-nav`. Use `.main-nav ul`, never `ul.main-nav`.
- `.toggled` is added to `#site-navigation` when the drawer is open — scope all drawer styles to this.

## 12. Footer

The footer is output by `mlk_render_footer()` in `render.php`, hooked to `generate_before_footer`. It's entirely custom PHP — not a GP Element or a widget area.

**Structure:**
```
.mlk-footer
  .mlk-footer__inner   [3-col grid]
    [brand name + address + social links]
    .mlk-footer__col   [Explore links]
    .mlk-footer__col   [Artists links]
  .mlk-footer__copy    [copyright line]
```

To add a new artist to the footer, edit `mlk_render_footer()` in `render.php` and add an `<a>` to the Artists column.

## 14. Known patterns & gotchas

### Hero height
The hero uses `min-height: 520px` via CSS. The `.mlk-hero__img` div (the `position:absolute` background image container) is always rendered even when there's no image, because removing it caused the hero to collapse. Don't remove it.

### ACF `get_field()` vs raw postmeta
`get_field('page_sections', $post_id)` returns a fully processed array — image fields become arrays with `url`, `alt` etc. Raw `get_post_meta()` returns the attachment ID. Always use `get_field()` in renderers.

### GP full-width reset
GP's `.grid-container` and `.site-content` have `max-width` set. These are reset to `none` in `style.css`:
```css
.site.grid-container  { max-width: none !important; padding: 0 !important; }
.site-content         { max-width: none !important; padding: 0 !important; display: block !important; }
```
If sections ever appear narrow/constrained, check these rules are loading.

### Two-col section — image as `<img>` not background
The two-col section renders images as `<img>` tags (via `mlk_render_image()`), not CSS `background-image`. This means:
- Images scale with the column naturally
- No extra `background-size`/`background-position` needed
- Add `aspect-ratio` or `height` CSS on `.mlk-twocol__img img` to control proportions

### Adding a new tint to a section
If you add a new `bg_tint` choice (e.g. `rust`) to a section, you must:
1. Add it to `fields.php` choices array
2. Add a CSS rule: `.mlk-[section]--rust { background: var(--rust); }`
3. Consider text contrast — rust/ink backgrounds need `color: var(--chalk)` on child elements

### WP_DEBUG
Currently `WP_DEBUG = true`, `WP_DEBUG_LOG = true`, `WP_DEBUG_DISPLAY = false` in `wp-config.php`. Turn `WP_DEBUG` off before deploying to production.

### data-populate.php in production
The `mlk_populate` URL trigger is gated by `current_user_can('manage_options')` — safe to leave in production. But don't run it on a production site unless you intend to overwrite page content.

### ACF field key underscore rows must match fields.php
If you rename a field key in `fields.php`, all existing `_page_sections_N_fieldname` postmeta rows will point to the wrong key. ACF will silently show empty. Fix: update the `_` prefixed rows in the DB, or use `update_field()` to re-save.
